# GitHub Upload Guide

This guide explains how to upload the project to GitHub and which files to include or exclude.

---

```
fruit_laplacian_segmentation/

├── README.md                            ✅ include
├── .gitignore                           ✅ include
```

---

## What NOT to Upload (Exclude)

| Path | Reason |
|------|--------|
| `data/fruits-360/` | ~2 GB dataset — too large for GitHub |
| `data/fruits-262/` | ~1 GB dataset — too large for GitHub |
| `outputs/models/*.pkl` | Trained model file — regenerated by running train script |
| `outputs/reports/` | Generated CSVs/PNGs — regenerated by evaluation |
| `outputs/examples/` | Generated example images — regenerated by evaluation |
| `outputs/segmentations/` | Generated overlays — regenerated by demo script |
| `__pycache__/`, `*.pyc` | Python cache files |
| `.venv/` | Virtual environment |

These are already listed in `.gitignore` — git will automatically skip them.

## Step-by-Step: First Upload

### 1. Install Git
Download from https://git-scm.com if not already installed.

### 2. Create a GitHub repository
1. Go to https://github.com and sign in.
2. Click **New repository**.
3. Name it: `fruit-laplacian-segmentation` (or any name you want).
4. Leave it **empty** (no README, no .gitignore — you already have these).
5. Click **Create repository**.

### 3. Initialize and push from your computer

Open a terminal (Command Prompt or PowerShell) and run these commands:

```powershell
# Go to the project folder
cd d:\mehdi\fruit_laplacian_segmentation

# Initialize git
git init

# Add all files (the .gitignore will automatically skip data/ and outputs/)
git add .

# Make the first commit
git commit -m "Initial commit: Fruit segmentation with Laplacian Pyramid (CV Assignment 2)"

# Link to your GitHub repo — replace YOUR_USERNAME and REPO_NAME
git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git

# Push to GitHub
git branch -M main
git push -u origin main
```

### 4. Verify on GitHub
Open your repository page on GitHub. You should see all the source files, but no `data/` contents or `outputs/` subdirectories.

## What To Tell Teammates (or the Professor)

After they clone the repo they need to:

1. **Clone the repo:**
   ```bash
   git clone https://github.com/YOUR_USERNAME/REPO_NAME.git
   cd REPO_NAME
   ```

2. **Install dependencies:**
   ```bash
   python -m venv .venv
   .venv\Scripts\activate          # Windows
   pip install -r requirements.txt
   ```

3. **Download the datasets manually** (Kaggle login required):
   - Fruits-360: https://www.kaggle.com/moltean/fruits → extract to `data/fruits-360/`
   - Fruits-262: https://www.kaggle.com/datasets/aelchimminut/fruits262 → extract to `data/fruits-262/`

4. **Run the full pipeline:**
   ```bash
   python scripts/train_5_fruits.py
   python scripts/evaluate_5_fruits.py
   python scripts/prepare_report_assets.py
   ```

## Uploading the LaTeX Report to Overleaf

The LaTeX report (`docs/project_report_for_teacher.tex`) is meant to be compiled on Overleaf.

1. Create a zip of the `docs/` folder:
   - On Windows: right-click `docs/` → Send to → Compressed (zipped) folder
2. Go to https://www.overleaf.com → New Project → Upload Project.
3. Upload the zip file.
4. Overleaf will find `project_report_for_teacher.tex` — click **Recompile**.
5. Make sure the `figures/` subfolder is present in the zip — the `.tex` file references `figures/confusion_matrix_5_fruits.png` etc.
